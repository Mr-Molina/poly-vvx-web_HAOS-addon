## 2024-05-24 - Unsanitized User Input in API URLs (SSRF / Path Traversal)
**Vulnerability:** The Flask application constructed external API requests by directly concatenating user-controlled configuration data (`app.config['SENSOR_ENTITY_IDS']`) into an API URL (`url + entity_id`). If an `entity_id` contains path traversal characters like `../` or arbitrary strings, it could lead to Server-Side Request Forgery (SSRF) or unintended data access. Additionally, unhandled exceptions related to network requests leaked full stack traces.
**Learning:** Even internal configuration values or localized settings must be treated as potentially unsafe input when constructing dynamic URLs for outgoing network requests. String concatenation for URLs is fragile and dangerous.
**Prevention:** Always use proper URL parsing/encoding libraries like `urllib.parse.quote(..., safe='')` to sanitize inputs before appending them to URLs. Ensure proper try-catch blocks are in place to mask internal errors from end-users, logging the details internally instead.
