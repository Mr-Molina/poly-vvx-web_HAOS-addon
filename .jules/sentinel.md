## 2026-03-07 - [Fail Securely on Upstream API Errors]
**Vulnerability:** The Flask application's API endpoint would crash the entire dashboard loop and return a 500 status code with a stack trace (in debug modes or uncaught exceptions) when an upstream Home Assistant API request failed for a single sensor.
**Learning:** This is a localized Denial of Service vulnerability on the Polycom VVX microbrowser. If one sensor fails to load (e.g. invalid entity ID or permissions issue), it breaks the state loop, meaning NO sensors are displayed to the user.
**Prevention:** Always implement per-item exception handling when iterating over multiple upstream requests. Ensure that individual failures are caught, logged, and return a safe, default "Error" state for that specific item, allowing the rest of the application to continue functioning securely without leaking implementation details or stack traces. Added a global exception handler to provide a final safety net against unhandled exceptions, returning a standard JSON error response instead of potentially leaking application state.
