## 2024-05-24 - Configuration Injection via `app.config.from_file`
**Vulnerability:** The Flask application used `app.config.from_file('/data/options.json', json.load)` to parse user-provided configuration. This method iterates through all keys in the JSON file and blindly injects them into the Flask `app.config` object. An attacker able to modify `options.json` could overwrite critical internal Flask settings like `DEBUG`, `TESTING`, or `SECRET_KEY`, potentially leading to Remote Code Execution (RCE) via the Werkzeug debugger or other information disclosure.
**Learning:** `app.config.from_file` and `app.config.from_mapping` should never be used with untrusted or user-supplied data, as they provide no filtering mechanism.
**Prevention:** Explicitly load the JSON file (`json.load`), extract only the explicitly expected configuration keys, and assign them individually to `app.config`. Always provide safe fallback values for missing keys to prevent unexpected application crashes.
